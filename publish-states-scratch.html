Overview Flow Wireframe Prototype Editor States

The editor header across five publishing states.

The header needs to communicate different things depending on where a page is in its publishing lifecycle. These mockups explore the visual language for each state — what the Publish button says, whether the Live badge shows, and what status text appears.

State 1 Never published

A brand new page. Nothing is live yet. The Publish button is the clear call to action — nothing to compare against, no Live badge to show.

← Back Times of Reston · Homepage ▾ Saved
No Live badge — nothing is live. Publish button is full-weight white: it's the only action that matters here.
State 2 Up to date

The page has been published and the editor hasn't changed anything since. Draft and live are identical. This is the calm resting state between publishes.

← Back Times of Reston · Homepage ▾ Last published 2h ago
✓ Live
Publish button is disabled — there is genuinely nothing to publish. The draft and live are identical, so the action has no meaning. Live badge is prominent. "Last published 2h ago" replaces the "Saved" text since the save status is no longer the relevant signal.
State 3 Unpublished changes

The editor has made changes since the last publish. The draft no longer matches what readers are seeing. This is the most important state to get right — the editor needs to know they have unfinished business.

← Back Times of Reston · Homepage ▾ ● Unpublished changes
✓ Live
Key decisions here: The amber "● Unpublished changes" indicator is the main signal — it replaces the quiet "Saved" text with something that demands attention. The Live badge stays green (the old version is still live, which is true). The Publish button label changes to "Publish changes →" to be specific about what will happen.
State 4 Publishing in progress

The editor has clicked Publish and the changelog modal animation is running. The modal is handling all the feedback — progress bar, button states, success confirmation. The header just stays out of the way.

← Back Times of Reston · Homepage ▾ ● Unpublished changes
✓ Live
The header looks the same as State 3 — the modal is on top of it anyway and the editor's attention is there. No progress bar in the header; the modal already has one. Publish button is locked (cursor: not-allowed) but the header doesn't change appearance. This state lasts about one second.
State 5 Just published

Publishing completed successfully. The draft and live are now identical again. This transitions back to State 2 (Up to date) after a short window.

← Back Times of Reston · Homepage ▾ Published just now
✓ Live
"Published just now" replaces the amber warning — the job is done. Publish button returns to its muted state. This is the same as State 2, just with a more immediate timestamp. After a few minutes it settles into "Last published X ago."
