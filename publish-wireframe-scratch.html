Publish button lives at the far right of the editor header — the primary action. Preview sits immediately to its left as a secondary action. After publishing, the button stays as "Publish" and a ✓ Live badge appears — clicking it opens the published page in a new tab. The badge is the persistent, always-visible link to the live version. The page title (▾) opens the page settings dropdown per KAN-8.

Lightweight modal — not a heavy confirmation dialog, just a low-friction prompt for a changelog note. The note is optional so editors aren't blocked. Clicking Publish here immediately puts the page live and starts the undo window.

C — Undo window (15 seconds after publishing)
← Page is live · Editor view continues normally above

Undo

Anchored to the bottom of the screen, not blocking anything. "View live page ↗" is right there for the most common post-publish action. The countdown is visible but quiet. Clicking Undo immediately reverses the publish. After 15 seconds the toast fades — the ✓ Live badge in the header takes over as the persistent link to the published page.

Page title is the natural home for page-level actions — clicking the name already signals "I want to do something with this page." Rename, settings, and duplicate are everyday actions. Revert to published sits below a divider in amber — visible but not casual; it only appears once the page has been published at least once. Delete is at the bottom in red, behind a confirmation dialog (KAN-6).
